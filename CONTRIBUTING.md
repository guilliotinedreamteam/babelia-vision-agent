# Contributing to Babelia Vision Agent

Thank you for your interest in contributing! This project aims to explore the boundaries of AI-powered pattern recognition in infinite combinatorial spaces.

## Ways to Contribute

### 1. Improve AI Models
- Better significance detection algorithms
- More efficient CLIP prompt engineering
- Advanced anomaly detection methods
- Alternative ML models for semantic understanding

### 2. Optimize Performance
- Faster image processing pipelines
- GPU acceleration improvements
- Parallel processing enhancements
- Memory optimization

### 3. Enhance Features
- Web dashboard for monitoring discoveries
- Real-time visualization
- Better sampling strategies
- Image classification refinements

### 4. Documentation
- Improve README
- Add tutorials and examples
- Document discoveries and findings
- Create video demonstrations

### 5. Testing
- Add unit tests
- Integration tests
- Performance benchmarks
- Edge case handling

## Development Setup

```bash
# Clone repository
git clone https://github.com/guilliotinedreamteam/babelia-vision-agent.git
cd babelia-vision-agent

# Create virtual environment
python -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
pip install -r requirements-dev.txt  # Development dependencies

# Run tests
pytest tests/ -v
```

## Code Style

- Follow PEP 8 guidelines
- Use type hints where applicable
- Write docstrings for all functions and classes
- Keep functions focused and modular
- Add comments for complex logic

## Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Add tests if applicable
5. Ensure all tests pass
6. Commit your changes (`git commit -m 'Add amazing feature'`)
7. Push to the branch (`git push origin feature/amazing-feature`)
8. Open a Pull Request

### PR Guidelines

- Provide a clear description of changes
- Reference any related issues
- Include test results if applicable
- Update documentation as needed
- Keep PRs focused on a single feature/fix

## Reporting Issues

When reporting bugs or suggesting enhancements:

- Use a clear, descriptive title
- Provide detailed steps to reproduce (for bugs)
- Include system information (OS, Python version, etc.)
- Attach relevant logs or screenshots
- Describe expected vs. actual behavior

## Discovery Reporting

If you discover significant images:

1. Save the image and metadata
2. Document the Babelia coordinates
3. Include analysis scores and reasoning
4. Open an issue with the "discovery" label
5. Attach the image (if appropriate)
